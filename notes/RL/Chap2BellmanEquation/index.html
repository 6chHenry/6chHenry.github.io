贝尔曼公式（强化学习）核心知识点总结

一、贝尔曼公式基础认知

  1. 核心定义：描述不同状态的状态价值（State Value，记为 vπ(s)v_\pi(s)vπ​(s)） 之间的关系，是计算状态价值的核心工具。
  2. 本质逻辑：将当前状态的价值拆分为“即时奖励”和“未来状态价值的折现”两部分，体现强化学习中“短期收益”与“长期收益”的权衡。

二、推导前置概念与公式

  1. 关键术语回顾
    • 折扣回报（Discounted Return，GtG_tGt​）：从时刻ttt开始的累计奖励，需考虑折扣因子γ\gammaγ（0≤γ≤10 \leq \gamma \leq 10≤γ≤1），公式为：Gt=Rt+1+γRt+2+γ2Rt+3+…G_t = R_{t+1} + \gamma R_{t+2} + \gamma^2 R_{t+3} + \dotsGt​=Rt+1​+γRt+2​+γ2Rt+3​+…
    • 状态价值（State Value，vπ(s)v_\pi(s)vπ​(s)）：在策略π\piπ下，从状态sss出发的折扣回报的期望，即vπ(s)=Eπ[Gt∣St=s]v_\pi(s) = \mathbb{E}_\pi[G_t | S_t = s]vπ​(s)=Eπ​[Gt​∣St​=s]
  2. 回报拆分关键步骤：将GtG_tGt​拆分为即时奖励和未来回报的折现，即Gt=Rt+1+γGt+1G_t = R_{t+1} + \gamma G_{t+1}Gt​=Rt+1​+γGt+1​，为后续期望拆分奠定基础。

三、贝尔曼公式推导过程

  1. 第一步：拆分期望（基于回报拆分）
    对状态价值公式中的期望进行拆分（期望的线性性质）：
    vπ(s)=Eπ[Rt+1+γGt+1∣St=s]=Eπ[Rt+1∣St=s]+γEπ[Gt+1∣St=s]v_\pi(s) = \mathbb{E}_\pi[R_{t+1} + \gamma G_{t+1} | S_t = s] = \mathbb{E}_\pi[R_{t+1} | S_t = s] + \gamma \mathbb{E}_\pi[G_{t+1} | S_t = s]vπ​(s)=Eπ​[Rt+1​+γGt+1​∣St​=s]=Eπ​[Rt+1​∣St​=s]+γEπ​[Gt+1​∣St​=s] 拆分后得到两项：即时奖励的期望和未来回报期望的折现。

  2. 第二步：计算”即时奖励的期望”
    需考虑策略π\piπ（状态sss下选择动作aaa的概率π(a∣s)\pi(a|s)π(a∣s)）和环境动态（动作aaa下获得奖励rrr的概率p(r∣s,a)p(r|s,a)p(r∣s,a)），公式为：
    Eπ[Rt+1∣St=s]=∑aπ(a∣s)E[Rt+1∣St=s,At=a]=∑aπ(a∣s)∑rr⋅p(r∣s,a)\mathbb{E}_\pi[R_{t+1} | S_t = s] = \sum_a \pi(a|s) \mathbb{E}[R_{t+1}|S_t=s,A_t=a] \\ =\sum_a \pi(a|s) \sum_r r \cdot p(r|s,a)Eπ​[Rt+1​∣St​=s]=∑a​π(a∣s)E[Rt+1​∣St​=s,At​=a]=∑a​π(a∣s)∑r​r⋅p(r∣s,a) 本质是“策略下所有动作的期望奖励加权和”。

  3. 第三步：计算”未来回报期望的折现”

    • 利用马尔可夫性质（无记忆性）：未来回报仅依赖于下一个状态s′s's′，与当前状态sss无关，即Eπ[Gt+1∣St=s]=Eπ[vπ(St+1)∣St=s]\mathbb{E}_\pi[G_{t+1} | S_t = s] = \mathbb{E}_\pi[v_\pi(S_{t+1}) | S_t = s]Eπ​[Gt+1​∣St​=s]=Eπ​[vπ​(St+1​)∣St​=s]。

    • 结合环境动态（动作aaa下转移到状态St+1s′S_{t+1}s'St+1​s′的概率p(s′∣s,a)p(s'|s,a)p(s′∣s,a)），公式为：

      \begin{align*} \mathbb{E}[G_{t+1}|S_t = s] &= \sum_{s'} \mathbb{E}[G_{t+1}|S_t = s, S_{t+1} = s']p(s'|s) \\ &= \sum_{s'} \mathbb{E}[G_{t+1}|S_{t+1} = s']p(s'|s) \\ &= \sum_{s'} v_\pi(s')p(s'|s) \\ &= \sum_{s'} v_\pi(s') \sum_{a} p(s'|s,a)\pi(a|s) \\ &=\sum_a\pi(a|s)\sum_{s'}p(s'|s,a)v_\pi(s') \end{align*}

  4. 最终贝尔曼公式
    合并上述两项，得到完整公式：
    vπ(s)=∑aπ(a∣s)[∑rr⋅p(r∣s,a)+γ∑s′p(s′∣s,a)⋅vπ(s′)]v_\pi(s) = \sum_a \pi(a|s) \left[ \sum_r r \cdot p(r|s,a) + \gamma \sum_{s'} p(s'|s,a) \cdot v_\pi(s') \right]vπ​(s)=∑a​π(a∣s)[∑r​r⋅p(r∣s,a)+γ∑s′​p(s′∣s,a)⋅vπ​(s′)]

四、贝尔曼公式的关键特性

  1. 多方程性：并非单一公式，而是对状态空间中所有状态sss 都成立。若有nnn个状态，会形成nnn个线性方程，联立可解出所有状态价值。
  2. 策略依赖性：公式中的π(a∣s)\pi(a|s)π(a∣s)（策略）决定了状态价值的计算结果，不同策略对应不同的状态价值（核心用途：策略评估（Policy Evaluation），即判断策略优劣）。
  3. 环境动态依赖性：依赖p(r∣s,a)p(r|s,a)p(r∣s,a)和p(s′∣s,a)p(s'|s,a)p(s′∣s,a)（环境模型/动态模型），若已知模型可直接计算；若未知模型，需用无模型强化学习（如蒙特卡洛、时序差分）方法。
  4. 自举性（Bootstrapping）：当前状态价值的计算依赖其他状态价值，需通过联立方程或迭代方法求解（而非直接计算）。

五、实例应用（基于视频案例）

案例1：单动作、确定转移的简单场景

  • 设定：状态s1s_1s1​下，策略π(a3∣s1)=1\pi(a_3|s_1)=1π(a3​∣s1​)=1（仅选动作a3a_3a3​）；动作a3a_3a3​的奖励r=0r=0r=0（概率1），转移到s3s_3s3​（概率1）。
  • 代入公式：
    即时奖励期望=0，未来回报期望=vπ(s3)v_\pi(s_3)vπ​(s3​)，最终贝尔曼公式简化为：vπ(s1)=0+γ⋅vπ(s3)v_\pi(s_1) = 0 + \gamma \cdot v_\pi(s_3)vπ​(s1​)=0+γ⋅vπ​(s3​)。
  • 直观结论：s1s_1s1​的价值仅依赖s3s_3s3​的价值，符合”离目标状态越近，价值越高”的逻辑（如视频中s2/s3/s4s_2/s_3/s_4s2​/s3​/s4​价值为10，s1s_1s1​价值为9=γ×109=\gamma \times 109=γ×10，γ=0.9\gamma=0.9γ=0.9）。

案例2：多动作、概率转移的场景

  • 设定：状态s1s_1s1​下，策略π(右∣s1)=0.5\pi(右|s_1)=0.5π(右∣s1​)=0.5、π(下∣s1)=0.5\pi(下|s_1)=0.5π(下∣s1​)=0.5；向右转移到s2s_2s2​（奖励-1），向下转移到s3s_3s3​（奖励0）。
  • 代入公式：
    vπ(s1)=0.5×(−1+γvπ(s2))+0.5×(0+γvπ(s3))v_\pi(s_1) = 0.5 \times (-1 + \gamma v_\pi(s_2)) + 0.5 \times (0 + \gamma v_\pi(s_3))vπ​(s1​)=0.5×(−1+γvπ​(s2​))+0.5×(0+γvπ​(s3​))。
  • 策略对比：该策略下s1s_1s1​价值为8.9（γ=0.9\gamma=0.9γ=0.9），低于案例1中s1=9s_1=9s1​=9的价值，说明”含惩罚动作的策略更差”。

六、公式的延伸意义

  1. 后续应用：通过贝尔曼公式计算出状态价值后，可进一步改进策略（如选择能提升状态价值的动作），最终逼近最优策略。
  2. 与无模型方法的关联：若未知环境模型（p(r∣s,a)p(r|s,a)p(r∣s,a)、p(s′∣s,a)p(s'|s,a)p(s′∣s,a)），贝尔曼公式的思想仍适用（如时序差分方法用经验估计期望），是强化学习的核心理论基础。

第2课-贝尔曼公式（公式向量形式与求解）知识点整理

一、贝尔曼公式的矩阵向量形式推导

1. 推导前提

  • 单条贝尔曼公式无法求解状态价值（State Value），因公式中同时包含当前状态价值与其他状态价值。
  • 对所有状态（共n个）而言，针对单个状态的贝尔曼公式（Elementwise Form）均成立，可通过整合n条公式得到矩阵向量形式，便于求解与理解。

2. 核心公式演化

  • 原始贝尔曼公式简化：将公式中“即时奖励项”与“状态转移概率项”合并，得到简化式：
    vπ(s)=rπ(s)+γ∑s′Pπ(s→s′)vπ(s′)v^\pi(s) = r^\pi(s) + \gamma \sum_{s'} P^\pi(s \to s') v^\pi(s')vπ(s)=rπ(s)+γ∑s′​Pπ(s→s′)vπ(s′)
    • rπ(s)r^\pi(s)rπ(s)：从当前状态sss出发，即时奖励（Immediate Reward）的平均值。
    • Pπ(s→s′)P^\pi(s \to s')Pπ(s→s′)：从状态sss转移到状态s′s's′的概率（基于策略π\piπ）。
  • 状态编号与向量定义：对所有状态标记为s1,s2,...,sns_1, s_2, ..., s_ns1​,s2​,...,sn​，定义以下向量与矩阵：
    • 状态价值向量 vπv^\pivπ：vπ=[vπ(s1)vπ(s2)...vπ(sn)]v^\pi = \begin{bmatrix} v^\pi(s_1) \\ v^\pi(s_2) \\ ... \\ v^\pi(s_n) \end{bmatrix}vπ=​vπ(s1​)vπ(s2​)...vπ(sn​)​​，包含所有状态的价值。
    • 即时奖励向量 rπr^\pirπ：rπ=[rπ(s1)rπ(s2)...rπ(sn)]r^\pi = \begin{bmatrix} r^\pi(s_1) \\ r^\pi(s_2) \\ ... \\ r^\pi(s_n) \end{bmatrix}rπ=​rπ(s1​)rπ(s2​)...rπ(sn​)​​，包含所有状态的即时奖励平均值。
    • 状态转移矩阵 PπP^\piPπ（State Transition Matrix）：
      矩阵元素[Pπ]ij[P^\pi]_{ij}[Pπ]ij​表示从状态sis_isi​转移到状态sjs_jsj​的概率，即[Pπ]ij=Pπ(si→sj)[P^\pi]_{ij} = P^\pi(s_i \to s_j)[Pπ]ij​=Pπ(si​→sj​)。
  • 最终矩阵向量形式：vπ=rπ+γPπvπv^\pi = r^\pi + \gamma P^\pi v^\pivπ=rπ+γPπvπ

3. 实例说明

实例1：确定性策略（4个状态）

  • 即时奖励向量rπr^\pirπ：
    从s1s_1s1​出发即时奖励为0，从s2,s3,s4s_2, s_3, s_4s2​,s3​,s4​出发即时奖励均为1，故rπ=[0111]r^\pi = \begin{bmatrix} 0 \\ 1 \\ 1 \\ 1 \end{bmatrix}rπ=​0111​​。
  • 状态转移矩阵PπP^\piPπ：
    第一行对应s1s_1s1​的转移概率：s1→s1s_1 \to s_1s1​→s1​（0）、s1→s2s_1 \to s_2s1​→s2​（0）、s1→s3s_1 \to s_3s1​→s3​（1）、s1→s4s_1 \to s_4s1​→s4​（0），即第一行为[0,0,1,0][0, 0, 1, 0][0,0,1,0]，其余行按策略同理推导。

实例2：概率性策略（4个状态）

  • 即时奖励向量rπr^\pirπ（以s1s_1s1​为例）：
    s1s_1s1​有0.5概率向右转移（奖励-1）、0.5概率向下转移（奖励0），故rπ(s1)=0.5×(−1)+0.5×0=−0.5r^\pi(s_1) = 0.5 \times (-1) + 0.5 \times 0 = -0.5rπ(s1​)=0.5×(−1)+0.5×0=−0.5。

    s1→s2s_1 \to s_2s1​→s2​（0.5）、s1→s3s_1 \to s_3s1​→s3​（0.5）、s1→s1s_1 \to s_1s1​→s1​（0）、s1→s4s_1 \to s_4s1​→s4​（0），即第一行为[0,0.5,0.5,0][0, 0.5, 0.5, 0][0,0.5,0.5,0]。

二、状态价值（vπv^\pivπ）的求解方法

1. 求解的核心意义

  • 该过程称为策略评估（Policy Evaluation），是强化学习的基础工具：只有通过评估策略的状态价值，才能判断策略优劣，进而改进策略、找到最优策略。

2. 两种求解方法

方法1：解析解（Closed-Form Solution）

  • 公式推导：基于矩阵向量形式vπ=rπ+γPπvπv^\pi = r^\pi + \gamma P^\pi v^\pivπ=rπ+γPπvπ，移项整理得：
    (I−γPπ)vπ=rπ(I - \gamma P^\pi) v^\pi = r^\pi(I−γPπ)vπ=rπ（III为单位矩阵），两边左乘(I−γPπ)−1(I - \gamma P^\pi)^{-1}(I−γPπ)−1，最终解析解为：
    vπ=(I−γPπ)−1rπv^\pi = (I - \gamma P^\pi)^{-1} r^\pivπ=(I−γPπ)−1rπ。
  • 局限性：仅理论优美，实际中因状态空间大时矩阵维度高，求逆计算量极大，难以应用。

三、策略评估的实例与结论

1. 实例背景

  • 奖励规则：尝试跳过边界/进入禁止区域（Forbidden Area）得-1，进入目标区域（Target Area）得+1，其他情况按策略定。
  • 折扣因子γ=0.9\gamma = 0.9γ=0.9。

2. 不同策略的评估结果

（1）优质策略

  • 策略特征：不跳边界、不撞墙、不进禁止区域，能导向目标区域。
  • 状态价值特征：所有状态价值均为正数；且离目标区域越近，状态价值越大（符合直觉，近目标区域更易获得高奖励）。
  • 特殊结论：不同策略可能得到相同状态价值（如某两个格子分别“向下走”和“向右走”，最终价值一致）。

（2）劣质策略

  • 策略1：所有状态均“向右走”（易撞墙/跳边界），状态价值全为负数。
  • 策略2：随机策略（多状态会撞墙/进禁止区域），状态价值与直觉一致（整体偏低）。

3. 核心结论

通过计算状态价值vπv^\pivπ，可直观判断策略的优劣：优质策略对应正的、较高的状态价值，劣质策略对应负的、较低的状态价值。

第2课-贝尔曼公式（Action Value的定义）知识点总结

核心概念：Action Value（动作价值）

1. 定义

  • 本质：智能体（Agent）从当前状态sss出发，选择某个动作aaa后，后续遵循策略πππ所能获得的平均回报（Average Return）。
  • 符号表示：qπ(s,a)q_π(s,a)qπ​(s,a)，其中：
    • πππ：表示当前遵循的策略，不同策略对应不同的动作价值；
    • (s,a)(s,a)(s,a)：状态-动作对，动作价值依赖于“从哪个状态出发”和“选择哪个动作”，是状态与动作的二元函数。

2. 核心作用

  • 动作价值是策略选择的核心依据：在某一状态下，智能体通过比较不同动作的qπ(s,a)q_π(s,a)qπ​(s,a)，优先选择价值更大的动作（该动作能带来更多长期回报）；
  • 后续强化学习算法（如Sarsa、Q-Learning）均以动作价值为核心展开，是实现策略优化的关键。

Action Value与State Value（状态价值）的关系

1. 两者区别与联系

维度State Value（状态价值）vπ(s)v_π(s)vπ​(s)Action Value（动作价值）qπ(s,a)q_π(s,a)qπ​(s,a)
定义从状态sss出发，遵循策略πππ的平均回报从状态sss出发、选择动作aaa后，遵循策略πππ的平均回报
依赖对象仅依赖状态sss和策略πππ依赖状态sss、动作aaa和策略πππ
核心作用评估状态的“好坏”评估“状态-动作对”的“好坏”，直接指导动作选择

2. 数学关联（核心公式）

（1）由Action Value推导State Value

状态价值是该状态下所有可能动作的动作价值，按策略πππ的动作选择概率加权平均的结果：
vπ(s)=Σa[π(a∣s)∗qπ(s,a)]v_π(s) = Σ_a [π(a|s) * q_π(s,a)]vπ​(s)=Σa​[π(a∣s)∗qπ​(s,a)]

  • π(a∣s)π(a|s)π(a∣s)：策略πππ下，在状态sss选择动作aaa的概率；
  • 含义：状态的价值由“该状态下所有可能动作的价值”及其“被选择的概率”共同决定。

（2）由State Value推导Action Value

结合贝尔曼公式（状态价值的递归表达式），可推导出动作价值的表达式：
qπ(s,a)=E[Rt+1+γ∗vπ(St+1)∣St=s,At=a]=∑rr⋅p(r∣s,a)+γ∑s′p(s′∣s,a)⋅vπ(s′)q_π(s,a) = E[R_{t+1} + γ * v_π(S_{t+1}) | S_t=s, A_t=a] = \sum_r r \cdot p(r|s,a) + \gamma \sum_{s'} p(s'|s,a) \cdot v_\pi(s')qπ​(s,a)=E[Rt+1​+γ∗vπ​(St+1​)∣St​=s,At​=a]=∑r​r⋅p(r∣s,a)+γ∑s′​p(s′∣s,a)⋅vπ​(s′)

  • Rt+1R_{t+1}Rt+1​：选择动作aaa后获得的即时奖励；
  • γγγ：折扣因子（衡量未来回报的重要性）；
  • St+1S_{t+1}St+1​：执行动作aaa后转移到的下一状态；
  • 含义：动作价值 = 即时奖励 + 折扣后的下一状态价值的期望；
  • 两者是“硬币的两面”：已知所有动作价值可求状态价值，反之已知所有状态价值也可求动作价值。

关键易错点与实例解析

1. 易错点：未被策略选择的动作，其Action Value是否为0？

  • 结论：否。即使策略πππ在状态sss下仅选择某一动作（如仅选a2a2a2），其他动作（如a1a1a1、a3a3a3）的动作价值仍需计算，而非0；
  • 原因：后续策略优化（如策略改进）需比较所有动作的价值，可能发现未被选择的动作实际价值更高，进而更新策略（选择该动作）。

2. 实例计算（确定性策略场景）

假设场景：状态s1s1s1有动作a2a2a2（策略指定动作）和a3a3a3（未被策略选择），转移规则如下：

  • 选择a2a2a2：即时奖励R=−1R=-1R=−1，转移到s2s2s2，vπ(s2)v_π(s2)vπ​(s2)已知；
  • 选择a3a3a3：即时奖励R=0R=0R=0，转移到s3s3s3，vπ(s3)v_π(s3)vπ​(s3)已知。

则动作价值计算为：

  • qπ(s1,a2)=−1+γ∗vπ(s2)q_π(s1,a2) = -1 + γ * v_π(s2)qπ​(s1,a2)=−1+γ∗vπ​(s2)（确定性转移，期望即确定值）；
  • qπ(s1,a3)=0+γ∗vπ(s3)q_π(s1,a3) = 0 + γ * v_π(s3)qπ​(s1,a3)=0+γ∗vπ​(s3)（即使未被策略选择，仍需按公式计算）。

Action Value的计算方法

  1. 间接法：先通过贝尔曼公式求解所有状态的vπ(s)v_π(s)vπ​(s)，再代入动作价值公式qπ(s,a)=E[Rt+1+γ∗vπ(St+1)]q_π(s,a) = E[R_{t+1} + γ * v_π(S_{t+1})]qπ​(s,a)=E[Rt+1​+γ∗vπ​(St+1​)]计算；
  2. 直接法：不依赖状态价值，直接通过数据（如轨迹采样）估算动作价值（后续蒙特卡洛、时序差分方法会详细讲解）；
  3. 两种方法均支持“基于模型”（已知状态转移概率和奖励函数）和“无模型”（未知环境模型，依赖采样数据）场景。
