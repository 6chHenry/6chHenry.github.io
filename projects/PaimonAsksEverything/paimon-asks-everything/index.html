Overview

它不只回答一个问题，而是连接一次版本发布的两端

玩家一端需要无剧透地接回复杂世界观，制作组一端需要知道大家究竟在哪里没看懂。PAIMON 把这两件事做成同一条回路：个性化预热建立语境，证据问答解决理解断点，关系图帮助继续探索，最小化匿名事件再汇总为发行简报与制作清单。

Problem

版本越大，真正稀缺的不是内容，而是被正确理解的路径

新玩家、回归玩家和剧情党需要不同的背景深度；统一导览既容易剧透，也容易漏掉关键上下文。

任务文本、官方前瞻、Wiki 与社区讨论混在一起时，流畅回答很容易把推测写成设定。

发行侧看到点击与问题数量，却不一定知道该补哪段时间线、谁最受影响、两周内能做什么。

Product Journey

四站任务线，把“看懂版本”变成可追踪的产品体验

这不是四个彼此独立的 Demo 页面。它们从建立上下文开始，经过查证、探索，再回到制作决策，形成一条双向的信息路线。

01 Context / Preheat

先认识旅行者，再决定讲到哪里

地区与身份不只改变标题，而是决定阵营入门、剧情回顾、时间线、证据和关系图的开放程度。

  • 八地区独立导览
  • 三套角色化视图
  • Level 3 剧透二次确认
02 Ask / Evidence

先查证，再让派蒙开口

问题先经过意图识别、别名扩展与剧透门控，再进入可切换的两条搜索链路；来源去重、分级并通过引用校验后才生成回答。

  • 自研检索 / 原生 Web Search
  • 真实 URL 与来源分级
  • 失败时一次有界回退
03 Explore / Graph

把角色与伏笔放回关系里

至冬关系图把角色、组织和概念做成可探索节点；节点详情、关系分析与文本线索共同指向证据等级，而不是只展示漂亮连线。

  • 多语言节点详情
  • 双节点关系分析
  • 文本线索溯源
04 Decide / Release

把匿名信号变成下一步动作

决策台不止汇总次数，而是给出玩家需求切片、机会与风险分数，以及带负责人、排期和验收标准的制作动作。

  • AI 中文制作组简报
  • 确定性机会 / 风险评分
  • 证据越界自动回退规则版

Evidence Chain

每个结论都要沿原路走回证据

默认路径从用户偏好开始，到证据回答结束；只有最小匿名信号会继续进入发行侧。模型不可用、引用越界或搜索不完整时，系统会显式降级，而不是用更自信的语气遮住失败。

  5. 05 来源治理

Reliability

可信度不是一句承诺，而是失败路径也经过设计

  • 官方、可信 Wiki、社区与未知网页分层处理，社区推测不会被包装成官方设定。
  • 无最终正文、来源无关、请求超时或截断时，每次请求最多回退一次，并清理无法映射的引用编号。
  • 没有模型密钥时仍可运行确定性证据回答；没有 Supabase 时切换到本地事件存储。
  • Vitest 固定覆盖剧透 token、双语检索、引用完整性、三类预热视图、图谱证据与发行规则。

Current State

已经上线，也把边界写在产品里

Overview

It connects both ends of a release, not just one question.

Players need to re-enter a dense world without being spoiled. Release teams need to know where understanding breaks. PAIMON turns both needs into one loop: personalized preheat establishes context, evidence Q&A resolves confusion, the relationship graph supports exploration, and minimal anonymous events become release briefs and production checklists.

Problem

As releases grow, the scarce resource is a path to correct understanding.

New, returning, and lore-focused players need different context depth; one briefing either spoils too much or explains too little.

Quest text, official previews, wikis, and community threads blur together, so fluent answers can quietly turn speculation into canon.

Counts alone do not tell a release team which timeline to clarify, who is affected, or what can ship in the next two weeks.

Product Journey

Four mission stops turn “understand the update” into a traceable experience.

These are not four disconnected demo pages. They establish context, verify claims, support exploration, and return evidence to production decisions as one two-way information route.

01 Context / Preheat

Know the traveler before choosing what to reveal.

Region and profile determine which faction primer, recap, timeline, evidence, and graph layers are actually shown.

  • Eight independent region guides
  • Three role-specific views
  • Second confirmation for level-three spoilers
02 Ask / Evidence

Verify first. Let Paimon speak second.

Intent, alias expansion, and spoiler gates precede two switchable search routes. Sources are deduplicated, graded, and citation-checked before generation.

  • Controlled search / native Web Search
  • Real URLs and source tiers
  • One bounded fallback on failure
03 Explore / Graph

Put characters and clues back into relationships.

The Snezhnaya graph turns characters, factions, and concepts into explorable nodes. Details, pair analysis, and text trails point back to evidence grades—not decorative lines.

  • Bilingual node details
  • Two-node relationship analysis
  • Traceable text clues
04 Decide / Release

Turn anonymous signals into the next production action.

The decision center goes beyond counts: player-demand slices, opportunity and risk scores, plus actions with owners, schedules, and acceptance criteria.

  • Chinese AI production brief
  • Deterministic opportunity / risk scores
  • Rule fallback when citations leave the whitelist

Evidence Chain

Every conclusion has to walk back to evidence.

The default path starts with player context and ends with an evidence answer; only a minimal anonymous signal continues to the release side. When models fail, citations leave the whitelist, or search results are incomplete, the system degrades explicitly instead of hiding failure behind confidence.

  1. 01 Profile + region
  2. 02 Topic orchestration
  3. 03 Controlled retrieval
  4. 04 Native search
  5. 05 Source governance
  6. 06 Citation validation
  7. 07 Evidence answer
  8. 08 Anonymous decision signal

Reliability

Trust is not a promise. Failure paths are designed too.

  • Official, trusted-wiki, community, and unknown-web sources are separated, so community theories do not become official canon.
  • Missing final text, irrelevant sources, timeouts, and truncation trigger at most one fallback; unmapped citation numbers are removed.
  • Without a model key the deterministic evidence answer still runs; without Supabase the event layer falls back to local storage.
  • Vitest covers spoiler tokens, bilingual retrieval, citation integrity, three preheat views, graph evidence, and release rules.

Current State

Live today, with its boundaries visible.

As of main@c55fc57, personalized preheat, both search routes, citation governance, the relationship graph, and the release decision center are live in the public demo. Known limits remain: Vercel cold starts, an external-trend layer still awaiting integration, process-local rate limiting, and the coverage ceiling of community wikis. The interface degrades clearly instead of hiding those constraints.
